Sign In & Singn Up System

We have using our simple logic customized sign in & up system. It basically implement all the fundamental functions such as: uniqueness check, data type check, identification check, etc. Some advanced functions such as: swap will require user sign in to use them.

Search Function

Our customized information retrieval algorithm is based on term-frequency, term-location, book rating and a user-selected topic. So the search results will ordering from the most relevant to the most irrelevant.

Tagging

We are using a background script to tagging all the books from our predefined tags pool. So the user could quickly figure out what is this book talking about, and this information will also be used in our recommendation system.

Book Details

We pull all the Book info from Google Books API, it literally returns all the books which fit the http request we have sent, we also set the book genre as "computer science" to make sure all the book we get is related to programming.

Swap

Want to trade off old stuff and get something different? Then use our book swap system. It allows users to list the books they have and want, then sending messages to each other in order to make the deal happens.

User Profile

Within the user profile we have a mailbox which receives all the messages, book recommendation list, favorite books archive, wish list, and trade off list.
